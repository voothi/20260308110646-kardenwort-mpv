# Design: Finalize Interaction Hardening

## Context
The Drum Window (Mode W) utilizes a unified mouse interaction factory (`make_mouse_handler`) and a dynamic binding system (`parse_and_bind`). Recent unification of these systems caused a conflict where informational actions (RMB tooltips) were inheriting behavioral traits of selection actions (dragging and focus movement). Furthermore, the interaction model needed to be hardened for remote control usage (Windows/Remote ghost-click suppression).

## Goals / Non-Goals

**Goals:**
- Implement strict behavioral isolation between "Selection" and "Action" mouse interactions.
- Harden the interface against hardware-induced ghost clicks using a smart temporal shield.
- Enable Mode W support for embedded subtitle streams without physical files.
- Maintain high responsiveness for modifier-based mouse combinations (Ctrl+Click, Shift+Click).

**Non-Goals:**
- Completely rewriting the Finite State Machine (FSM).
- Adding complex multi-touch or gesture support.

## Decisions

### 1. The `updates_selection` Opt-Out
The `make_mouse_handler` factory now accepts an explicit `updates_selection` boolean. 
- **True (Default)**: Executes Phase 2 (setting `DW_CURSOR_LINE/WORD` and `DW_ANCHOR_LINE/WORD`) and Phase 3 (enabling `DW_MOUSE_DRAGGING`).
- **False (Action-specific)**: Skips selection logic. This allows RMB to pin tooltips while the previously active yellow focus remains unmoved.

### 2. The `MOUSE_HANDLERS` Registry
Since Lua functions cannot contain custom fields directly, a local `MOUSE_HANDLERS` weak table is introduced to track functions generated by the factory. This allows `parse_and_bind` to detect pre-wrapped handlers and avoid redundant "double-wrapping" which was causing state flicker.

### 3. Smart `nav` Wrapper (Mouse Shield)
Keyboard and remote navigation keys are wrapped in a `nav` decorator that sets `FSM.DW_MOUSE_LOCK_UNTIL = current_time + 150ms`. 
- **Universality**: Applied to all static (Arrows, Enter) and dynamic config-defined keyboard keys.
- **Modifier exclusion**: Standalone presses of `Ctrl`, `Shift`, `Alt`, or `Meta` are excluded from triggering the shield, ensuring that "Modifier + Click" combos work instantly without the 150ms lockout.

### 4. Memory-First Track Validation
Initialization logic in `cmd_toggle_dw` is updated to check `#Tracks.pri.subs == 0` if `path` is nil. This enables support for internal subtitle tracks which are pushed to the engine via memory rather than external disk files.

## Risks / Trade-offs

- **Temporal Shield**: If a user tries to click *extremely* fast after an arrow key (under 150ms), the click will be ignored. This is considered an acceptable trade-off for the stability provided to remote control and touchpad users.
- **Lua Environment**: Using a tracking table for functions is safer than attempting metatable hacks on functions in the Restricted-Lua/LuaJIT environment of `mpv`.
