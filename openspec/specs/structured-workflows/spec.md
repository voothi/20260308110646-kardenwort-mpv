# Spec: Structured Workflows

## Context
Standardizing the development lifecycle improves predictability and quality.

## Requirements
- Define a "Propose" phase: Create a new change with proposal, design, and specs.
- Define an "Apply" phase: Implement tasks based on the specs.
- Define an "Archive" phase: Move completed changes to a permanent archive and update master specs.

## Verification
- Verify that development follows the Propose → Apply → Archive sequence.
- Confirm that transition between phases is documented and reproducible.
