# Spec: Agent Capabilities Documentation

## Context
Users and agents need a clear understanding of what specialized tools and workflows are available.

## Requirements
- Create an `AGENTS.md` file in the project root.
- Document specialized skills (e.g., `openspec-propose`, `openspec-apply`).
- Document slash commands and their purposes.
- Maintain a clear mapping between user commands and agent actions.

## Verification
- Check for `AGENTS.md` in the root directory.
- Verify that documented slash commands match the implementations in `.agent/workflows/`.
