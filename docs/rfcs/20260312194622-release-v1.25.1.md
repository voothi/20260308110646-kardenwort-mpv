# RFC: Release v1.25.1 (Compact Proximity Search)

**ZID**: 20260312194622
**Date**: 2026-03-12
**Status**: STAGED

## Goal
Address "false positive" matches in fuzzy keyword search where letters are scattered across an entire subtitle line. The goal is to prioritize "Compact" matches where fuzzy letters are concentrated within the same word or adjacent words.

## Implementation Details

### 1. Match Span Calculation
- **Utility**: Upgraded `is_fuzzy_subsequence` to `find_fuzzy_span()`.
- **Logic**: Instead of a boolean, it now returns the `start` and `end` indices of the fuzzy match. 
- **Metric**: The engine calculates `span = end - start + 1`.

### 2. Compactness Bonus (Idea A++)
- **Comparison**: The engine compares the `span` to the original token length.
- **Scoring**:
    - **Ultra-Compact (+150)**: Match span is almost equal to query length (likely within a single word).
    - **Compact (+50)**: Match span is slightly larger but still localized.
- **Rationale**: This pushes results like `mne` matching "**m**a**n**ag**e**" (Span: 6) above `mne` matching "**m**...**n**...**e**" across three different words (Span: ~50).

### 3. Order Stability
- **Decision**: Maintained the sequential order bonus. Even with order-independent search, matches that follow the text's flow are rewarded to maintain intuitive ranking.
