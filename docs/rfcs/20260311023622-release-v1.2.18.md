## Title: Release v1.2.18 - Advanced Mouse Selection & Navigation

### Overview
This release completely overhauls the Drum Window's user interaction model by introducing intuitive, text-editor style mouse selection. The feature relies on a new unified layout engine to guarantee pixel-perfect hit-testing across complex, multi-line wrapped subtitles. Additionally, the release solidifies the Drum Window's environment by explicitly managing native OS window interactions and overlapping subtitle layers.

### Implementation Details

#### Unified Layout Engine & Hit Testing
- **Problem**: Previously, `draw_dw` relied on mpv/ASS's automatic word-wrapping (`\q0`). This made reverse-engineering the exact coordinates of wrapped words mathematically impossible, breaking hit-testing on long subtitles.
- **Solution**: Developed `dw_build_layout(subs, view_center)` to calculate hard visual line breaks using proportional font-width estimation. 
- **Rendering**: The `draw_dw` renderer was refactored to use this layout table, bypassing ASS smart-wrapping via the `\q2` tag. What you click is now algorithmically guaranteed to be exactly what you see.
- **Aggressive Snapping**: `dw_hit_test` was enhanced to snap to the nearest word if clicking outside the exact bounding box. This prevents the script from dropping mouse `down` events that slightly miss the text.

#### Hardware-Accelerated Dragging
- **Problem**: A 50ms polling timer introduced severe input lag when dragging to select text rapidly.
- **Solution**: Dragging now dynamically binds to the mpv `mouse_move` hardware event. The selection highlight tracks the cursor seamlessly at 60fps+. The 50ms timer is now strictly dedicated to edge-proximity auto-scrolling.

#### OS/Native Conflict Resolutions
- **Window Dragging Hijack**: Single-click-and-drag was failing because mpv natively interprets clicks on the video area as instructions to move the entire application window (which absorbs the event). The script now temporarily sets `window-dragging=no` when the Drum Window opens, returning to normal on close.
- **Subtitle Overlap**: The Drum Window (with its dark beige background) would often clash with native subtitle layers or the Drum Mode (c) overlay. `cmd_toggle_drum_window` now snapshots the visibility state of all tracks, forces them off during reading mode, and cleanly restores them upon exit.

#### Advanced Workflows
- **Double-Click to Seek**: A new `MBTN_LEFT_DBL` binding allows users to instantly seek playback to the exact `start_time` of any visible subtitle in the Drum Window. This automatically centers the viewport and re-enables "Follow Player" mode.
- **Point-to-Point Selection**: Refined the `Shift+MBTN_LEFT` logic to respect previously established anchor points (via click or keyboard navigation), allowing users to extend their selection accurately without dragging.
- **Scroll Synchronization**: Added `Ctrl+UP`/`Ctrl+DOWN` to match mouse wheel scrolling. Modified the `UP`/`DOWN` arrow navigation so that if the cursor wanders off-screen via scrolling, the first arrow press aggressively snaps the viewport back to the cursor instead of drifting further away.

### Analytics
These features represent a high-intensity, iterative refinement phase focusing heavily on precision UI mathematics and mpv's low-level input handling.
