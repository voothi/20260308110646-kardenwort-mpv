# RFC: Release v1.26.2 (Externalized Search HUD Styling)

**ZID**: 20260312212143
**Date**: 2026-03-12
**Status**: STAGED

## Goal
Decouple Search HUD visual presentation from core script logic by introducing user-configurable parameters. This allows for precision tuning of colors and bolding independent of future script updates.

## Implementation Details

### 1. Configuration Parameters (Options Table)
Introduced six new parameters to the `Options` table to control every aspect of Search HUD highlighting:
- **`search_hit_color`**: BGR hex for matching characters in result lists.
- **`search_hit_bold`**: Boolean toggle for bolding search hits.
- **`search_sel_color`**: BGR hex for the selected entry text.
- **`search_sel_bold`**: Boolean toggle for bolding the entire selected entry.
- **`search_query_hit_color`**: BGR hex for hits/selection in the top input field.
- **`search_query_hit_bold`**: Boolean toggle for bolding input field hits.

### 2. UI Rendering Logic (Analytics)
- **Dynamic Formatting**: Updated `draw_search_ui` to substitute hardcoded ASS tags with variable-driven formatting strings.
- **State Sensitivity**: The highlighting engine now correctly differentiates between "selected hits" and "unselected hits," allowing for complex contrast scenarios (e.g., white text on a red background).
- **Consolidation**: This release also incorporates the **Ultra-Minimalist** aesthetic (removal of markers and legacy red-overrides) while providing the architectural hooks to restore them via config if desired.
