# RFC: Release v1.2.0 - FSM Architecture Overhaul

**ZID**: 20260310002147

## Objective
Migrate the MPV Language Learning Suite scripts into a unified Finite State Machine (FSM) architecture to harmonize all operating modes, eliminate internal race conditions, and drastically improve code reliability and maintainability.

## User Request
*   **Request**: "Think about our switches, for better consistency and reliability, and ease of maintenance, does it make sense to rewrite using this programming pattern (FSM). This would make it possible to harmonize all operating modes, depending on the type and number of loaded subtitles. Reliably, while simplifying the code."

## Implementation Details (Analytics & Decisions)
1.  **Consolidation of Ad-Hoc Scripts**: 
    Prior to this refactoring, functionality such as `autopause.lua`, `sub_context.lua`, and `copy_sub.lua` operated iteratively with independent timer loops (`mp.add_periodic_timer`) and tracked state asynchronously via scattered local boolean flags. This led to occasional collisions and complex state dependencies. They were carefully replaced by merging their core features into a unified `scripts/lls_core.lua` file.
    
2.  **State Definition (`MEDIA_STATE`)**:
    We introduced an overarching analytical state tracker that determines the exact operating context directly from the native MVP `track-list` property: `NO_SUBS`, `SINGLE_SRT`, `SINGLE_ASS`, `DUAL_SRT`, `DUAL_MIXED`, `DUAL_ASS`. Every feature now structurally branches off this single source of truth, creating perfectly predictable state transitions based on the type of active subtitles.

3.  **Unified Master Tick Loop**: 
    Rather than spinning three uncoordinated plugin clocks, all runtime processing (Drum Mode calculations and Autopause tracking) now execute within a singular master tick loop at `0.05` ticks. This mathematically mitigates processing collisions and immediately enforces feature boundaries—for example, safely auto-disabling Drum Mode if the `MEDIA_STATE` strictly bounds against complex `.ass` matrices.

4.  **Preserved Keybinding Architecture**: 
    The original script-messages and command signatures were identical mapped within the FSM structure, ensuring the user's carefully curated dual-layout `input.conf` seamlessly interoperates with the new `lls_core.lua` without demanding configuration rewrites.
