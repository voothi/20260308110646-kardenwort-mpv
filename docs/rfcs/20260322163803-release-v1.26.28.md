# RFC: Release v1.26.28 - Search Box Visibility Fix (OSD Styling)

**ZID**: 20260322163803
**Date**: 2026-03-22

## Overview

This release fixes a visual regression where the `Ctrl+f` search box (and the Drum Window) became unreadable when using the "Black Frame" subtitle style (`background-box`). The system now intelligently overrides the global OSD border style only while these custom UI components are active.

## User Request

After implementing a black border for the subtitle font (`sub-border-style=background-box`), it looks like this change affected the Ctrl F search box. We need to make sure that it remains readable in both modes.

## Implementation Details

### Analytics & Decisions

- **Conflict Identification**: `mpv`'s `osd-border-style=background-box` draws a semi-transparent black box behind ALL OSD text. Since the Search UI already uses a custom beige background panel, this resulted in ugly, overlapping black boxes that obscured the text and results.
- **Dynamic Override**: Instead of trying to fight the global style with ASS tags (which `mpv` often overrides), the most robust solution is to temporarily change the `osd-border-style` property itself while the script's custom UI is on-screen.
- **Reference Counting**: Implemented a state-aware logic (`manage_ui_border_override`) that ensures the original user style is only restored when BOTH the Search UI and the Drum Window are closed. This prevents flickering or premature restoration if one is closed while the other is still active.

### Technical Implementation

- **`lls_core.lua`**:
    - Added `saved_osd_border_style` to the Finite State Machine (`FSM`).
    - Created `manage_ui_border_override(enable)` to encapsulate the save/override/restore logic.
    - Integrated this helper into `manage_search_bindings` and `cmd_toggle_drum_window`.
    - The override forces `outline-and-shadow` (the standard look) which provides clean text on top of the beige script panels.

## Verification Results

- **Styling Parity**: Confirmed that search box text is now perfectly clear on the beige background even when "Mode 1" (`background-box`) is active in `mpv.conf`.
- **Restoration**: Verified that closing the search box immediately restores the "Black Frame" look for standard OSD messages (like volume changes), preserving the user's aesthetic preference for the rest of the player experience.
- **Concurrent UI**: Verified that the override persists correctly if Search is opened on top of the Drum Window and then closed.
