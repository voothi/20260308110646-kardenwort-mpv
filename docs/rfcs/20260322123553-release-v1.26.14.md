# 20260322123553 - Release v1.26.14

## Objective
Finalize release v1.26.14, focusing on fixing a critical edge case where the first subtitle of an external `.srt` file was skipped in Drum Mode due to an unhandled UTF-8 Byte Order Mark (BOM).

## Implementation Details

### Analytics & Root Causes
- **UTF-8 BOM Interference**: Many `.srt` files begin with a 3-byte Byte Order Mark (`\xEF\xBB\xBF`). The custom subtitle parser in `lls_core.lua` used a strict digit match (`^%d+$`) to identify the start of a subtitle block (the ID).
- **First Subtitle Omission**: Because the very first line of a BOM-encoded file is actually `[BOM]1` rather than just `1`, the pattern match failed. The parser then skipped the entire first block until it encountered a blank line and correctly identified the second subtitle (`2`).

### Decisions & Solutions
- **BOM Stripping**: Updated the `clean_text_srt()` helper function to proactively strip the UTF-8 BOM sequence from the start of any line before processing.
- **Robustness**: This change ensures the first subtitle is correctly parsed regardless of whether the file uses BOM or not, without affecting the rest of the parsing logic or regular subtitle display.
