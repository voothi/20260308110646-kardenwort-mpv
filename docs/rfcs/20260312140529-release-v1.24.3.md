# RFC - v1.24.3 (Advanced Search Selection & Highlights)

**ZID**: 20260312140529
**Date**: 2026-03-12

## Objective
Implement VSCode-style word-by-word selection and visual highlighting within the Universal Search overlay.

## Implementation Details
1.  **Selection Anchor State**: Introduced `SEARCH_ANCHOR` to the `FSM` state. This tracks the starting point of a selection range. If `SEARCH_ANCHOR` is `-1`, no selection is active.
2.  **Word Boundary Detection**: Added a `get_word_boundary(q_table, pos, direction)` utility that uses pattern matching (`[%w\128-\255]`) to identify the boundaries of words in both English and Cyrillic (UTF-8).
3.  **Advanced Navigation Bindings**:
    - `Shift + Left/Right`: Extends the selection character-by-character.
    - `Ctrl + Left/Right`: Jumps the cursor to the previous/next word boundary.
    - `Ctrl + Shift + Left/Right`: Extends the selection word-by-word.
4.  **Selection-Aware Editing**:
    - Typing or pasting (`Ctrl+V`) while a selection is active now automatically deletes the selected range before inserting new text.
    - `Backspace` and `Delete` keys now clear the entire selected range if one exists.
5.  **Dynamic Highlight Rendering**: Updated the `draw_search_ui` OSD engine to dynamically inject ASS color tags (`{\1c&H...&}`) around the selection range (the slice between `SEARCH_ANCHOR` and `SEARCH_CURSOR`). This provides clear visual feedback of the selected text.

## Modified Files
- `scripts/lls_core.lua` - logic for `SEARCH_ANCHOR`, word boundaries, selection deletion, and highlight rendering.
