# RFC: Release v1.2.6 - Keybinding Source of Truth Consolidation

**ZID**: 20260310024112

## Objective
Consolidate all keybinding definitions into `input.conf` as the single source of truth by removing the last hardcoded key (`"c"`) from `scripts/lls_core.lua`. Additionally, cleaned up the repository by removing obsolete files and improving `.gitignore`.

## User Request
*   **20260310023646**: *"Can you somehow reorganize the keyboard shortcuts so that they are all put into one source of truth - the input.conf file? Now something may still be in scripts, etc."*

## Implementation Details

### Analytics
An audit of all `mp.add_key_binding` calls across the `scripts/` directory revealed exactly **one offender**: line 768 of `lls_core.lua` hardcoded `"c"` as the default key for `toggle-drum-mode`. All 10 other bindings already used `nil` (no default key), correctly deferring to `input.conf`.

Meanwhile, `input.conf` already contained the line `c script-binding toggle-drum-mode`, making the hardcoded `"c"` in the script redundant and potentially conflicting.

### Decision
Replaced the hardcoded `"c"` with `nil` in `mp.add_key_binding(nil, "toggle-drum-mode", cmd_toggle_drum)`. This ensures:
- All 11 script bindings now use `nil` — purely named command handlers.
- `input.conf` is the **exclusive** authority for which physical keys map to which commands.
- Changing any keybinding requires editing only one file.

### Repository Cleanup
To further stabilize the build environment:
- **Git Cache Optimization**: Removed `scripts/old_copy_sub.lua` from git tracking (ZID 20260310024333). This obsolete standalone script is now correctly ignored as its functionality is fully merged into `lls_core.lua`.
- **Ignore Patterns**: Updated `.gitignore` to include `__pycache__/`, preventing redundant Python bytecode files from cluttering the working directory.
