# RFC - v1.24.0 (Universal Subtitle Search)

**ZID**: 20260312115025
**Date**: 2026-03-12

## Objective
Implement a high-performance, standalone subtitle search system with fuzzy-matching, clipboard integration, and mouse support to facilitate rapid text-based navigation during immersion.

## Implementation Details
1.  **Universal Search Architecture**: Decoupled the search mechanism from the Drum Window. Created a dedicated `search_osd` overlay that functions independently, allowing users to summon the search bar directly from standard playback.
2.  **Robust Input Handling**: Implemented a query buffer that captures raw keyboard input. Included multi-byte UTF-8 support for Cyrillic characters to ensure the Russian layout is a first-class citizen. Added precise Backspace logic for multi-byte deletion.
3.  **Clipboard Integration**: Utilized PowerShell (`Get-Clipboard`) to bridge the gap between mpv and the system clipboard (mapped to `Ctrl+V` and `Ctrl+М`). This allows users to paste external materials directly into the search bar.
4.  **Synchronized Playback Jumps**: Migrated all subtitle-based navigation from lightweight `time-pos` property assignments to explicit `seek absolute+exact` commands. This forces the media engine to synchronously update both primary and secondary track anchors, eliminating desynchronization when jumping from search results.
5.  **Mouse Interaction Engine**: Implemented an OSD-coordinate based hit-test for the dropdown list items. Users can now use `MBTN_LEFT` to select search results directly from the overlay.
6.  **Boolean Visibility Preservation**: Fixed a recurring bug in the Drum Window's close logic where default boolean fallbacks (`or true`) were inappropriately overriding a user's decision to keep subtitles hidden.

## Modified Files
- `scripts/lls_core.lua` - core search state machine, OSD rendering, input capture, and sync fixes.
- `input.conf` - Global `Ctrl+F` and `Ctrl+А` bindings.
