# RFC - v1.2.14 (Terminology Standardization & Goal Refinement)

**Date**: 2026-03-10
**Request ZID**: 20260310145832
**Status**: STAGING

## Executive Summary
This release completes the pivot of the project's identity from a "Language Learning" tool to a "Language Acquisition" suite. It standardizes all documentation and metadata to reflect the philosophy of **Extensive Acquisition** through immersion. The project goals have been refined to emphasize its role in the convenient consumption of Dual-Subtitle (DualSubs) content, especially for local offline workflows.

## Implementation Details

### 1. Terminology Standardization
Comprehensive audit and update of the project's "dictionary". The term "Language Learning" has been replaced with **"Language Acquisition"**, and "Study" has been replaced with **"Immersion"** or **"Acquisition"** throughout the codebase and documentation.
- **Scope**: `README.md`, `mpv.conf`, `release-notes.md`, `analyze_repo.py`, and standard reports.

### 2. Project Goals Refinement (Consumption Focus)
The `README.md` now explicitly states that the suite's mission is the **convenient consumption** of Dual-Subtitle material, rather than the "transformation" of video content. 
- **Key Use Case**: Handling original vs. translated subtitle tracks for acquisition.
- **Workflow Clarity**: Separated the preparation phase (pre-processing with external tools) from the playback phase (immersion using this suite).

### 3. YouTube Auto-Subtitle Context
Documented the suite's specialized capabilities (Drum Mode, Smart Spacebar) in mitigating context loss when acquiring language from lower-quality or unsynchronized YouTube auto-captions.

## Proposed Changes
- **README.md**: Updated goals, TOC, and standardized terminology.
- **mpv.conf**: Standardized comments for language acquisition.
- **release-notes.md**: Standardized terminology for both current and historical entries.
- **analyze_repo.py**: Updated internal metadata for the Acquisition Suite.
