# RFC - v1.24.1 (Search Navigation & Cursor Control)

**ZID**: 20260312133948
**Date**: 2026-03-12

## Objective
Implement VSCode-style text navigation and editing within the Universal Search overlay, allowing users to move the cursor, insert text, and delete characters at arbitrary positions.

## Implementation Details
1.  **UTF-8 Character Array Logic**: Migrated the `SEARCH_QUERY` manipulation from raw string concatenation to a mid-layer UTF-8 character array. This ensures that navigation and deletion are position-aware and byte-safe for multi-byte Cyrillic characters.
2.  **Moveable Cursor State**: Introduced `SEARCH_CURSOR` (0 to N) to track the current insertion point. `LEFT` and `RIGHT` arrow keys move this pointer.
3.  **Position-Aware Editing**: 
    - **Insertion**: Typings and clipboard pastes now use `table.insert` at the exact cursor position.
    - **Deletion**: Refactored `BS` (Backspace) to use `table.remove` at the cursor position. Added a new binding for the `DEL` key to remove the character immediately following the cursor.
4.  **Visual HUD Cursor**: Updated the OSD rendering engine to inject a vertical bar (`|`) at the calculated cursor position. Added a subtle transparency effect for the "Search..." placeholder text when the field is empty.
5.  **Layout Redefinition**: Temporarily hijacked `LEFT`/`RIGHT` keys during Search Mode to prevent 2-second track seeks, redirecting them to query navigation instead.

## Modified Files
- `scripts/lls_core.lua` - logic for `SEARCH_CURSOR`, UTF-8 utilities, and re-binding arrow keys.
