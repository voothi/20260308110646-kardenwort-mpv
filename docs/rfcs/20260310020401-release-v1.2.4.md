# RFC: Release v1.2.4 - Drum Sync & Compatibility Guards

**ZID**: 20260310020401

## Objective
Finalize the release of v1.2.4, addressing the keybinding regressions in Drum Mode identified after the v1.2.0 FSM migration and implementing a comprehensive suite of "Smart Compatibility Guards" to prevent feature misuse in mismatched subtitle environments.

## User Requests & ZID Audit Trail

    *   **Context**: Discovery that `j` and `y` keys were behaving erratically when Drum Mode was active.
*   **20260310021444**: *"Why [...] in drum mode in srt subtitles the `y` option does not work for me, but it works without the drum."*

*   **20260310022604**: *"Make Secondary Sub Pos work only in simple srt subtitle mode [...] Make Copy Subtitle Mode only work for .ass files."*
    *   **Context**: Request for sanity checks to prevent trying to reposition ASS tracks or filter pure SRT tracks.

## Implementation Details

### 1. The "Tick Override" Fix (Syncing commands with FSM)
During the v1.2.0 FSM rewrite, we introduced a 50ms `master_tick` loop. We discovered that `tick_drum` was reading native mpv properties (`secondary-sub-pos`) to calculate stacking. When the user pressed `y`, the property changed, but within 50ms, the tick loop re-read the value and immediately recalculated/overwrote it, making the keypress seem ignored.
- **Decision**: Introduced `FSM.native_sec_sub_pos` to store the *desired* state. All "Drum OFF" and "Shutdown" paths now correctly restore this value to mpv.

### 2. The "Ghost Subtitle" Fix (Stale Array Flushing)
We found that `update_media_state` was not clearing the internal `Tracks.subs` arrays when a track was disabled (SID=0) or changed.
- **Decision**: Added path-change detection. If the subtitle file path changes (or becomes `nil`), the memory array is immediately flushed. This allows `j` to correctly "hide" subtitles in Drum Mode.

### 3. Smart Compatibility Guards
Implemented a guard layer in `scripts/lls_core.lua` to catch invalid states:
- **`y` (Secondary Pos)**: Disables if no secondary sub exists or if it's an ASS track (which handles its own positioning).
- **`Ctrl+Z` (Copy Mode)**: Only enables for ASS or dual-track configurations where language filtering is mathematically possible.
- **`Ctrl+X` (Context Copy)**: Now strictly requires external subtitle file paths (as it must read from disk).
- **`c` (Drum Mode)**: Now explicitly blocks when no subtitles are loaded.
