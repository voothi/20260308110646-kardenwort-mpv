# RFC - v1.2.16 (Drum Window Evolution & Static Reading Mode)

**Date**: 2026-03-11
**Request ZID**: 20260311014935
**Status**: STAGING

## Executive Summary
This release marks a significant evolution of the Drum Window, transforming it from a simple scrolling list into a powerful **Static Reading Mode**. It introduces sophisticated viewport management that allows users to independently navigate and select text without interference from video playback. The selection mechanism has been upgraded to support multi-line ranges and text-editor style navigation.

## Implementation Details

### 1. Static Reading Mode (Viewport Decoupling)
The Drum Window now supports two distinct operational states: **Follow Mode** (defaulting to playback) and **Manual Mode** (triggered by any navigation). 
- In Manual Mode, the viewport is completely "frozen," providing a static, flicker-free environment for reading and selection. 
- The viewport only scrolls when the navigation cursor hits the top or bottom boundaries (edge-scrolling).

### 2. Multi-line & Substring Selection
Upgraded the navigation system to support range selection using `Shift`.
- **Shift + LEFT/RIGHT**: Word-by-word substring selection.
- **Shift + UP/DOWN**: Multi-line selection across subtitle boundaries.
- **Refined Copy**: `Ctrl+C` now aggregates all selected words across multiple lines into a single, clean clipboard export.

### 3. Navigation & Consistency
- **Initial Pointer Position**: Upon opening, the word pointer is automatically positioned at the first word of the current subtitle.
- **Seek Synchronization**: Pressing `a`/`d` (seek) now clears any active selection and re-enables Follow Mode, ensuring the viewport always stays in sync with playback after a manual jump.
- **Dual-Layout Robustness**: Full mappings for Russian keyboard layouts (`Shift + ВВЕРХ/ВНИЗ/ЛЕВЫЙ/ПРАВЫЙ`) ensuring a consistent experience for all users.

### 4. Visual & Layout Polishing
- Implemented `\q0` subtitle wrapping to prevent long lines from overlapping.
- Optimized line spacing (unified `\N`) to maximize visible context.
- Fixed the "white trail" highlight bug to ensure visual precision during rapid navigation.
