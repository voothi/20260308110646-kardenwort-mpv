# RFC: Release v1.26.4 (Cyrillic Import Fix & UI Silence)

## Problem Description
1. **Cyrillic Import Bug**: When importing `.ass` subtitle files, Cyrillic characters (likely from translation tracks) were appearing in the Drum Window's primary subtitle area, cluttering the English reading experience.
2. **UI Noise**: Toggling the Drum Window produced status messages ("Drum Window: OPEN/CLOSED") that felt redundant and distracting during intensive immersion sessions.

## Implementation Details

### Analytics & Decisions
1. **Hoisting for Global Scope**:
   - **Action**: Hoisted `has_cyrillic`, `is_word_char`, and `build_word_list` to the top of the file.
   - **Decision**: Ensures these core utilities are initialized before any parsing functions, resolving potential initialization order issues and standardizing text processing throughout the script.
2. **Hardening via Nil-Safety**:
   - **Action**: Implemented guards in all base text functions (`if not str then return false end`).
   - **Decision**: Strategic defense-in-depth to prevent runtime exceptions when processing malformed or empty subtitle lines.
3. **Targeted .ass Filtering**:
   - **Action**: Injected a `has_cyrillic` check into the `Dialogue:` line parser within `load_sub`.
   - **Decision**: Minimally invasive way to filter out translation noise at the source. It ensures the Drum Window only contains target-language (non-Cyrillic) content for the primary track.
4. **OSD Suppression**:
   - **Action**: Commented out the "OPEN/CLOSED" OSD messages in `cmd_toggle_drum_window`.
   - **Decision**: Prioritized a minimalist, "premium" feel. The visual state of the window is immediate feedback.

### Verification Results
- **Regression Check**: Compared `2e4d8a0` to `051ea8b`. Verified that the diff is 100% focused on the intended hoisting, filtering, and OSD changes.
- **Manual Verification**: Confirmed that `.ass` files with mixed tracks no longer show Cyrillic lines in `w` mode.
