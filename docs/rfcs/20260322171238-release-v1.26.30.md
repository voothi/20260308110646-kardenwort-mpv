# RFC: Release v1.26.30 - Search Selection Fix (Scoping Bug)

**ZID**: 20260322171238
**Date**: 2026-03-22

## Overview

This release fixes a critical regression in the Search Mode (`Ctrl+f`) where attempting to select text using `Ctrl+Shift+Arrow` would cause the script to crash and stop responding. The root cause was a Lua scoping issue where a utility function was called before its local definition.

## User Request

Selecting a line in the search field starting from the position of the installed carriage, to the beginning of the word, has stopped working, similar to how it works in the VSCode text editor. It worked before. Now I call Ctrl F on the main screen, a search window appears, I type any text, then press Ctrl Shift left arrow or Shift left arrow and it’s as if the script crashes. The windows close, there are no elements on the screen. It is no longer possible to call w or Ctrl f in the current session. I use q to exit.

## Implementation Details

### Analytics & Decisions

- **Scoping Crash**: In Lua, `local` functions must be defined before they are used if they are referenced at the same lexical level. The `get_word_boundary` function (responsible for word-based navigation) was attempting to call `is_word_char`, but `is_word_char` was defined later in the file. This resulted in a `nil` function call at runtime, crashing the entire script engine for the session.
- **State Integrity**: While investigating the search crash, a similar field-name mismatch was found in the Drum Window's word-selection logic (`FSM.ANCHOR_LINE` vs `FSM.DW_ANCHOR_LINE`). Although not explicitly reported, this was fixed to ensure robust multi-word selection.

### Technical Implementation

- **`lls_core.lua`**:
    - Reordered utility functions: `is_word_char` is now defined before `get_word_boundary`.
    - Corrected state field references in `cmd_dw_word_move` to match the `DW_` prefix pattern used in the rest of the script.

## Verification Results

- **Selection Stability**: Confirmed that `Ctrl+Shift+Left/Right` now correctly highlights words in the search query without triggering script errors.
- **Persistence**: Verified that the script remains active after selection actions, and other features like Drum Window (`w`) continue to work as expected.
