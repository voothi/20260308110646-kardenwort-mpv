## Overview
This release marks the transition of the experimental "Fixed Font" scaling logic into a first-class feature within the core `lls_core.lua` engine. It introduces a refined, mathematically balanced scaling algorithm and centralized configuration management.

## Implementation Details

### 1. Smart Font Scaling Port
Moved the subtitle scaling logic from the standalone `fixed_font.lua` into `lls_core.lua`. This was done to ensure a single source of truth for all subtitle-related logic and to simplify the user's installation.

### 2. The "Softer Scaling" Formula
A significant issue was identified with "strict" font locking: at very small window heights, forcing a 1080p font size results in aggressive word wrapping (4-5 lines), which obscures the video. 

**Decision**: Implement a linear interpolation formula (`scale_strength`) that allows the user to tune the balance.
- **Formula**: `comp_scale = 1.0 + (perfect_comp - 1.0) * Options.font_scale_strength`
- **Logic**: This provides "weighted" compensation. At `0.5` strength, the font shrinks exactly halfway as fast as it normally would. This maintains readability without exploding the text layout into massive vertical blocks on small windows.

### 3. Centralized `mpv.conf` Control
Added formal `script-opts` to the project's configuration authority. Users can now enable/disable the feature and fine-tune its behavior without editing Lua code.

### 4. Codebase Cleanup
Deleted `scripts/fixed_font.lua`. The core engine now natively handles `osd-dimensions` and `track-list` property observation to update scaling in real-time.
