# 20260310000706 - Release v1.1.0 (ASS Context Copy Enhancements)

## Implementation Details
The user reported a critical bug involving Context Copy (`Ctrl + X`) failing on complex `.ass` files. Specifically, files with interleaved dual-language tracks (e.g., English and Russian) and word-by-word highlighted karaoke styling caused Context Copy to loop over fragments of the exact same line, failing to fetch true surrounding conversational context.

### Decisions & Analytics
- **Root Cause Analysis:** The original Lua parsing logic in `copy_sub.lua` identified karaoke duplicates by checking only the immediately prior subtitle line (`subs[#subs].raw_text == raw_text`). Because a dual-track `.ass` file alternates between an English line and a Russian translation, this strict sequential check failed. Every karaoke fragment was treated as a distinct dialog block. Thus, requesting "2 sentences ago" merely returned the 2 previous word-tokens of the current sentence.
- **Improved Karaoke Merging:** Refactored the array loading parser to search up to backward up to 10 entries to merge identical raw text lines. This bridges over interleaved translations, successfully reassembling entire chronological sentences.
- **Language-Aware Context:** `get_context_text` was rewritten to fetch targeted contexts, dynamically continuing its search backwards and forwards in the array until it satisfies the `context_copy_lines` quota, deliberately ignoring translations or irrelevant tracks using Cyrillic characteristic detection.
