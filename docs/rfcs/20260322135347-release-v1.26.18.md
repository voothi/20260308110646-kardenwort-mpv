# RFC: Release v1.26.18 (Centralized Config & Styling)

**Request ZID**: 20260322135347
**Date**: 2026-03-22
**Status**: DRAFT → PROPOSED

## Overview
This release focuses on centralizing script configuration and standardizing the visual styling of the package's configuration files (`mpv.conf` and `input.conf`). By moving all script parameters into the main configuration and unifying the documentation style, we've made the suite more accessible and easier to manage from a single source of truth.

## Included Change Requests (ZIDs)
- **20260322132952**: Migration of all `lls_core.lua` parameters into `mpv.conf`.
- **20260322134235**: Consistency and uniform styling audit for configuration files.
- **20260322134928**: Final functional verification and restoration of useful configuration templates.

## Implementation Details

### 1. Centralized Script Configuration
Relocated all adjustable parameters from the `Options` table in `scripts/lls_core.lua` into the main `mpv.conf` file using the `script-opts-append` syntax.
- **Benefit**: Users can now fully customize the behavior of AutoPause, Drum Mode, Copy Mode, and Search HUD without modifying the underlying Lua code.
- **Stability**: The script retains its original `Options` table as a hardcoded fallback, ensuring stability even if the `mpv.conf` entries are deleted or malformed.

### 2. Styling Standardization
Unified the visual language of both `mpv.conf` and `input.conf`:
- **Header Width**: Standardized all major section headers to a 75-character width using the `# ===================` style.
- **Subsection Markers**: Implemented the `# --- Title ---` format for all subsections.
- **Vertical Spacing**: Normalized spacing between properties and sections to improve file scannability and professional feel.

### 3. Documentation & Template Restoration
Restored and standardized useful configuration templates (e.g., language priorities, visibility toggles, scaling overrides) as commented-out examples. These are now explicitly labeled as `(Template)` to guide the user in common customization tasks.
