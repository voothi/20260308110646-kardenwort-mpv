# RFC: Release v1.26.0 (Search Hit Highlighting)

**ZID**: 20260312202316
**Date**: 2026-03-12
**Status**: STAGED

## Goal
Implement "elegant" visual feedback for search queries by highlighting the specific characters that matched the query within the search results dropdown.

## Implementation Details

### 1. Character-Level Indexing
- **Refactor**: Replaced `find_fuzzy_span` (range-based) with `find_fuzzy_indices` (list-based).
- **Metadata**: `calculate_match_score` now returns an `all_indices` table identifying every character index that contributed to the fuzzy match.
- **Caching**: Search result objects in `FSM.SEARCH_RESULTS` now store `{idx, hl}`, where `hl` is the bit-map of matched indices.

### 2. Dynamic Contrast Rendering
- **States**:
    - **Normal**: White text with **Bold Red** hits.
    - **Selected**: Red background/text with **Bold White** hits.
- **Logic**: Injected ASS tags (`\b1`, `\c&H...&`) iteratively during OSD display string construction.
- **Optimization**: Added a 120-character "Visual Truncation" limit for result line rendering to ensure OSD performance remains high even with heavy tagging.
