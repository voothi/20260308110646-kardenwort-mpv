# RFC: Release v1.26.22 (Drum Window Hit-Test Calibration)

**Request ZID**: 20260322153215
**Date**: 2026-03-22
**Status**: DRAFT → PROPOSED

## Overview
This release addresses a regression in the Drum Window hit-testing accuracy introduced by increasing the font size to 34. It implements configurable hit-test multipliers for vertical line height, subtitle gaps, and character width, allowing for precise mouse selection even at large font scales.

## Included Change Requests (ZIDs)
- **20260322142455**: Initial report of deteriorated click accuracy at `dw_font_size=34`.
- **20260322153215**: Finalization of calibration settings and release creation.

## Implementation Details

### 1. Configurable Hit-Test Multipliers
Added the following options to `lls_core.lua` to decouple the hit-test grid from hardcoded font size ratios:
- `dw_vline_h_mul`: Adjusts the calculated height of visual lines.
- `dw_sub_gap_mul`: Adjusts the vertical gap between subtitle blocks (`\N\N`).
- `dw_char_width`: Adjusts the horizontal width per character for monospace hit-testing.

### 2. Multi-Mode Configuration Structure
Introduced a "Mode" structure in `mpv.conf` similar to the existing subtitle modes. This allows users to quickly switch between different font size calibrations (e.g., swapping between 30 and 34).

### 3. Calibrated Values (Consolas)
Based on visual analysis and user testing, the following values were established as the new defaults for `dw_font_size=34`:
- **vline_h_mul**: 0.87
- **sub_gap_mul**: 0.6
- **char_width**: 0.5
