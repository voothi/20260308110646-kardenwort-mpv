# RFC - v1.2.22 (Track Scrolling Keys)

**ZID**: 20260311101023
**Date**: 2026-03-11

## Objective
Add additional keyboard shortcuts (`Shift + A`, `Shift + D`, and their Russian layout counterparts) to map directly to the existing exact 2-second track scrolling keys (`LEFT` and `RIGHT` arrows).

## Implementation Details
1. **Navigating without Layout Switching**: The user requested that shortcuts work efficiently in EN/RU layouts without having to constantly switch inputs.
2. **Context on Window Mode (w)**: In normal mode, scrolling tracks by 2 seconds works natively with the arrow keys. However, when the Drum Window (Static Reading Mode) is active, arrow keys are hijacked to scroll the text within the window viewport. 
3. **The Solution**: By introducing `Shift+A` and `Shift+D` (`A` and `D` mapped to `seek -2 exact` and `seek 2 exact`), the user can now scroll 2 seconds back and forth while the window mode is active. This avoids conflicting with the navigational arrow keys and ensures that track scrolling is always accessible regardless of the active viewing mode.
4. **Russian Layout Equivalents**: Mapped `Ф` and `В` (which correspond to `Shift+A` and `Shift+D` on a Cyrillic keyboard) to perform the exact same 2-second seeking actions. 

## Modified Files
- `input.conf` - Added `A`, `D`, `Ф`, `В` exact seek bindings.
