# RFC: Release v1.26.26 - Cross-Platform Clipboard Support

**ZID**: 20260322161222
**Date**: 2026-03-22

## Overview

This release introduces native cross-platform clipboard support, removing the hard dependency on Windows PowerShell for copying and pasting subtitles. The system now automatically detects the operating system and uses the appropriate command-line utilities for clipboard operations.

## User Request

Ensure the operation of all functions not only on Windows 11, but also on MacOS, Linux, and Android. Specifically focusing on platform-dependent parts like copying selected substrings.

## Implementation Details

### Analytics & Decisions

- **Platform Detection**: Used `package.config:sub(1,1)` to distinguish between Windows (`\`) and Unix-like systems (`/`).
- **Unified Abstraction**: Created `get_clipboard()` and `set_clipboard(text)` to encapsulate the platform-specific logic, simplifying the rest of the codebase.
- **Utility Selection**:
    - **Windows**: Retained PowerShell `Get-Clipboard`/`Set-Clipboard` for best compatibility.
    - **macOS**: Leveraged standard `pbcopy` and `pbpaste`.
    - **Linux**: Added support for both Wayland (`wl-copy`/`wl-paste`) and X11 (`xclip`/`xsel`).
    - **Android (Termux)**: Integrated `termux-clipboard-get`/`set` to support mobile immersion setups.

### Technical Implementation

- Refactored `cmd_dw_copy`, `cmd_copy_sub`, and `paste_from_clipboard` in `lls_core.lua` to use the new unified helper functions.
- Removed redundant string formatting and escaping logic from the main command functions, centralizing it in the `set_clipboard` helper.

## Verification Results

- Verified the Windows implementation continues to function correctly.
- Structure confirmed to correctly branch for non-Windows platforms based on `uname` results.
