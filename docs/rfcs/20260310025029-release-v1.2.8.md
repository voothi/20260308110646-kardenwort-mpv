# RFC: Release v1.2.8 - Hotkey Simplification & Documentation

**ZID**: 20260310025029

## Objective
Enhance the user experience by simplifying core hotkeys, reorganizing the input configuration structure, and providing comprehensive inline documentation for all controls.

## User Requests & ZID Audit Trail

*   **20260310025029**: Replaced `Ctrl+X` with `X` and `Ctrl+Z` with `Z`. Simplified keys to work without modifiers and across keyboard layouts.
*   **20260310025343**: Reorganized `input.conf` into logical groups (Navigation & System, Language Layouts, Feature Toggles).
*   **20260310025511**: Added descriptive comments for every combination in `input.conf` to explain the "what" and "why" behind each binding.

## Implementation Details

### Analytics
The previous modifier-heavy keys (`Ctrl+X`, `Ctrl+Z`) were identified as friction points for rapid study. Furthermore, the lack of documentation in `input.conf` made the suite's advanced features (like "Smart Spacebar" or "Drum Mode") less discoverable for new users or after long breaks.

### Decisions
1.  **Single-Key Shortcuts**: Moved `toggle-copy-context` to `x`/`X` and `cycle-copy-mode` to `z`/`Z`. Added capital-letter variants and Russian layout counterparts (`ч`/`я`) to ensure functionality regardless of `Shift` or system language.
2.  **Structural Grouping**: Grouped keys by functional domain. This highlights the dual-layout symmetry (EN vs RU) and separates core navigation (arrows/space) from specific study features (toggles).
3.  **Inline Instructions**: Every keybinding now has a comment explaining its specific behavior within the language learning context (e.g., explaining that `LEFT` is fixed at 2 seconds to avoid overshooting phrases).

## Verification Plan
1.  **Hotkey Test**: Verified that pressing `x` (lowercase) or `X` (uppercase) triggers context copy.
2.  **Layout Test**: Verified that `я` (Russian) correctly cycles copy mode.
3.  **Visual Audit**: Confirmed `input.conf` is now neatly formatted and fully commented.
