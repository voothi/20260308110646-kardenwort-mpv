# RFC: Release v1.24.10 (Search Relevance & Cyrillic Parity)

**ZID**: 20260312185338
**Date**: 2026-03-12
**Status**: STAGED

## Goal
Solve the "chronological bias" in search results where the first matching line in the file was always prioritized at the top, regardless of match quality. Also, ensure full case-insensitive matching for Cyrillic characters.

## Implementation Details

### 1. Relevance Scoring Engine
- **Logic**: Each potential match is assigned a numeric score:
    - **Exact Match** (case-insensitive): 1000 points.
    - **Prefix Match** (substring at index 1): 800 points.
    - **Substring Match** (found anywhere): 500 points.
    - **Fuzzy/Subsequence Match**: 100 points.
- **Sorting**: `update_search_results` now collects all matches into a temporary table and uses `table.sort` (Score DESC, Index ASC) before finalizing the visible results list.
- **Rationale**: Ensures the most user-relevant results (the ones they are likely looking for) appear first, even if they appear late in the subtitle file.

### 2. Cyrillic Case Mapping
- **Problem**: Lua standard `string.lower` is ASCII-only and fails to normalize Cyrillic upper/lower cases.
- **Solution**: Implemented `utf8_to_lower(str)` utility using a predefined mapping of standard Russian UTF-8 character pairs.
- **Rationale**: Restores "Presence" of records that were previously ignored due to capitalization mismatches in Russian subtitles.
