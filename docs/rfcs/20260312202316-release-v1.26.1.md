# RFC: Release v1.26.1 (High-Contrast Search Aesthetics)

**ZID**: 20260312202316
**Date**: 2026-03-12
**Status**: STAGED

## Goal
Simplify the Search HUD visual language by removing colored highlights and adopting a "Black & Bold" high-contrast design. This improves legibility and provides a more professional, terminal-like aesthetic.

## Implementation Details

### 1. High-Contrast Highlighting
- **Logic**: Search "hits" (matching characters) are now highlighted using and only using the **Bold** (`{\b1}`) ASS tag.
- **Color Removal**: Removed `dw_highlight_color` (Red) from all search match rendering.
- **Background Integrity**: The selected line no longer inverts colors or uses a specific highlight hue, maintaining the "Black on Beige" consistency throughout the dropdown.

### 2. Selection Feedback
- **Marker**: Added a subtle `> ` marker for the selected line to provide navigation feedback without the need for color-based highlighting.
- **Rationale**: Adheres to the user's request for "no highlighting in any way except bold search entries" while still allowing for unambiguous navigation.
