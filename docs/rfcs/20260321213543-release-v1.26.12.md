# 20260321213543 - Release v1.26.12

## Objective
Finalize release v1.26.12, focusing on perfecting Drum Mode rendering, unifying subtitle layouts, and cleaning up OSD elements.

## Implementation Details

### Analytics & Root Causes
- **Drum Box Overlaps**: Previously, Drum Mode drew individual background boxes per line due to ASS tag injection. This caused tight lines to bleed padding onto each other.
- **Top Subtitle Bifurcation**: Drum Mode traditionally pushed top subtitles upwards and active subtitles downwards from a central coordinate, causing a noticeable vertical gap.
- **Positioning Sync Bugs**: Toggle keys (`y`) and Ctrl+T/R in normal mode adjusted mpv properties, but Drum Mode ignored live properties in favor of hard-coded settings (snapping statically to 10% or 90%), preventing adjustments from taking visual effect.
- **Double Newline Gap**: A logic error in string concatenation appended `\N\N` instead of `\N` inside Drum Mode text blocks, causing empty heights between the historical, active, and future contexts.
- **Ugly OSD Scale**: Subtitle jumps (`a`/`d`) invoked the default mpv OSD bar, cluttering the screen unnecessarily when the user prefers the elegant OSC via `TAB`.

### Decisions & Solutions
- **Unified Tagging**: Centralized Drum ASS tags to wrap the entire grouped block (prev, active, next), removing internal box bleeding.
- **Unified Anchors**: Abandoned the split `\an8` / `\an2` gap logic. Top contexts now form one seamless block anchored via `\an8`, and bottom contexts run seamlessly via `\an2`.
- **String Fix**: Cleaned the concatenation pipeline to ensure exactly one `\N` string separates contexts, destroying the artificial blank gaps.
- **Live Sync**: Bound `cmd_cycle_sec_pos` to also directly update the mpv `secondary-sub-pos` property, and told `tick_drum()` to derive its visual Y coordinate straight from `mp.get_property_number`.
- **Decoupled Configuration**: Forced `osd-border-style=outline-and-shadow` internally for Drum Mode in `mpv.conf`, leaving `background-box` purely for standard subtitles where it shines.
- **Disabled `osd-bar`**: Uncommented `osd-bar=no` in `mpv.conf`, gracefully hiding the built-in time scale so it only appears strictly via OSC toggle (`w` or `TAB`).
