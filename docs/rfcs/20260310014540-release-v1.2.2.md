## Technical Narrative

### What Happened
During the massive migration to the `v1.2.0` FSM architecture, the Context Copy loop was rewritten to dynamically skip translated Cyrillic tracks. When the user complained that the output contained Russian text (caused by a separate bug in the Ternary Operator), we mistakenly assumed the *entire dynamic loop array logic* was flawed and rolled the system mathematically backward to the "simple" `v1.1.0` blind-radius loop (`idx -2 to idx +2`).

However, as the user demonstrated in **ZID 20260310010508** and **ZID 20260310013255**, this simple strict-radius logic is geographically mathematically destructive when evaluating densely interleaved dual-track `.ass` files.

### What Helped
We executed direct Python simulations dumping the internal memory arrays of `load_sub` at the precise `time_pos` keystrokes.
This revealed two fundamental mathematical truths about the array:
1.  **Strict Radii are Flawed**: Grabbing a strict 5-index mathematical radius `[idx-2 to idx+2]` from an array that perfectly alternates `[English, Russian, English, Russian]` guarantees the script harvests 3 Russian strings and 2 English strings. Ass-filtering then deletes the Russian strings, leaving the user with only 2 fragmented English sentences. The dynamic jumping loop from `v1.2.0 FSM` was actually structurally superior all along.
2.  **Center-Index Symmetry Collapse**: We discovered a secondary anomaly where `get_center_index` (due to dual-tracks sharing exact timestamps) would randomly target the *Russian* translation line as the center pivot. The filter loop would then skip adding a center string entirely, reducing the chronological output from a 5-sentence request down to a 4-sentence request (`2 Pre + 0 Center + 2 Post`).

### The Result
We deployed a comprehensive, two-part algorithmic redemption in `scripts/lls_core.lua`:

1.  **Restored Dynamic Traversal**: The Context Copy look-back/look-forward arrays once again dynamically execute a `while` loop that bounds completely over explicitly filtered tracks until the strict mathematical English sentence quota is fulfilled (e.g., retrieving precisely 2 actual English strings backwards, regardless of how many Russian strings intercept them).
2.  **Target Language Index Snapping**: We injected an override into the `get_center_index` resolution block. If the algorithm randomly pivots on a filtered foreign-language block, it now instantly peeks at `idx+1` and `idx-1`. Upon detecting the native English block sharing the identical timestamp, it snaps the `idx` pivot onto the correct natively-targeted block before building the mathematical arrays.

These two changes permanently guarantee flawless, symmetrical, chronological Context string extraction from interleaved `.ass` tracks, solving all missing-sentence reports. Finally, the `is_context` clipboard pipeline shortcut identified in commit `45e8ae320` was successfully restored to reduce redundant parsing overhead.
