# RFC: Release v1.26.32 - Improved Drum Window Navigation & Pointer Logic

**ZID**: 20260322191027
**Status**: Staging
**Release**: v1.26.32

## Summary
This release focuses on refining the user experience in the Drum Window by deactivating the word-level pointer by default and improving the reliability of subtitle navigation (a/d keys), especially when paused after an autopause.

## Proposed Changes

### 1. Pointer Behavior Refinement
- **Default State**: The Drum Window now opens with the word pointer (`DW_CURSOR_WORD`) set to `-1` (inactive). No word is highlighted in red initially, allowing for immediate full-line or context copying without interference.
- **Auto-Deactivation**: Scrolling with the mouse wheel or jumping to a new subtitle via search now automatically resets the pointer to `-1`.
- **Manual Activation**: The pointer "wakes up" and highlights the first word only when the user explicitly uses the arrow keys (`UP`, `DOWN`, `LEFT`, `RIGHT`).

### 2. Navigation Reliability (Double-Tap Fix)
- **Problem**: The native `sub-seek` command sometimes failed to jump to the next subtitle on the first press if the player was paused near the end of a subtitle (typical for autopauses).
- **Solution**: Implemented a custom seeking function `cmd_dw_seek_delta` that uses the internal subtitle table to find the target line and performs an `absolute+exact` seek. This ensures immediate response on every press.

## Implementation Details

### Analytics & Decisions
- **Consistency**: Decided to treat search selection identically to window opening to prevent accidental word-level copying right after a jump.
- **Robustness**: Replaced the native `sub-seek` with local logic because the player's internal state during pause-at-padding thresholds was causing inconsistent behavior. Using the internal `Tracks.pri.subs` table is more reliable as it's already pre-loaded into memory while the window is active.

## Verification Results
- Verified that `Ctrl+c` copies the full line immediately after opening `w`.
- Verified that `d` jumps to the next sub on the first press after an autopause.
- Verified that arrow keys correctly activate the word pointer.
