## Goal
Enhance the user experience of the **Search HUD** by adding professional-grade keyboard navigation and deletion shortcuts, specifically native support for **Bash-style Word Deletion** (`Ctrl+W`).

## Implementation Details

### 1. Bash-Style Deletion (`Ctrl+W`)
- **Action**: Bind `Ctrl+W` (English) and `Ctrl+Ц` (Russian) to `delete_word_before_cursor()`.
- **Decision**: Logic utilizes the existing `get_word_boundary` utility to identify the start of the word/whitespace block to the left of the cursor.
- **Selection Sensitivity**: If an active text selection is present, `Ctrl+W` prioritizes deleting the selection range (Standard UI behavior). If no selection is active, it deletes the preceding word.
- **Rationale**: Provides parity with terminal environments, allowing power users to quickly correct search queries without repeated backspaces.

### 2. Dual-Layout Parity
- **Decision**: Mapped `Ctrl+W` and `Ctrl+Ц` to ensure functionality is layout-agnostic.
- **Rationale**: Maintains the project's core philosophy of first-class Cyrillic support.
