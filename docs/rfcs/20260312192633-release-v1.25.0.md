# RFC: Release v1.25.0 (True Fuzzy Keyword Search)

**ZID**: 20260312192633
**Date**: 2026-03-12
**Status**: STAGED

## Goal
Evolve the search engine from a sequential subsequence matcher to a **True Fuzzy Keyword Matcher** (inspired by `fzf` and Bash). The goal is to allow users to find subtitles by typing non-contiguous keywords in any order, while each keyword itself can be a fuzzy approximation.

## Implementation Details

### 1. Tokenized Fuzzy Logic (Idea A+)
- **Mechanism**: The search query is split into whitespace-separated tokens.
- **Per-Token Matching**: Each token must exist within the target subtitle as either:
    1. A literal substring (High Signal).
    2. A fuzzy subsequence (Approximate Signal).
- **Order Independence**: Keywords can appear in any order in the text (e.g., typing `fox quick` matches `The Quick Brown Fox`).
- **Rationale**: This matches the mental model of most modern interactive search tools.

### 2. Multi-Dimensional Scoring
To ensure results remain intuitive, a tiered scoring system is used:
- **Base (500)**: All keywords found.
- **Literal Bonus (+100 per token)**: Rewarding exact character sequences within words.
- **Sequential Bonus (+300)**: Rewarding keywords that follow the file's text order.
- **Proximity/Substring Bonus (+400)**: Rewarding the exact literal phrase matches.
- **Start-of-Line Bonus (+300)**: Prioritizing sentences starting with the first keyword.

## Analytics Verdict
- **Logic Complexity**: Increased (nested matching loops), but optimized via early-exit for unmatched tokens.
- **User UX**: Significant improvement in search "forgiveness"—typos and word-order swaps no longer break the search.
