# RFC - v1.2.10 (Centralized Configuration & Safety Guards)

**Date**: 2026-03-10
**Request ZID**: 20260310120822
**Status**: STAGING

## Executive Summary
This release focuses on centralizing script behavior control into `mpv.conf` and implementing critical safety guards to prevent subtitle overlapping. It resolves a regression where primary and secondary subtitles would "stick together" when toggled to the same position.

## Implementation Details

### 1. Centralized Script Configuration
Implemented `mp.options` support in `lls_core.lua`. This allows users to override internal script parameters (`sec_pos_top`, `sec_pos_bottom`) directly from `mpv.conf` without modifying the Lua source code.

### 2. The "Safety Gap" Logic
Through analytical debugging of commit `5afa865` vs `a2915ae`, it was determined that a 100% dynamic positioning strategy (linking secondary position exactly to primary `sub-pos`) caused a collision at 95% Y-coordinate.
- **Decision**: Reintroduced explicit `sec_pos_bottom` (90) which maintains a **5% vertical gap** from `sub-pos` (95).
- **Documentation**: Added cross-referencing `[LINKED]` tags in `mpv.conf` to warn users about this technical dependency.

### 3. Positional Robustness
Replaced strict semantic equality checks (`== 10`) with threshold-based logic (`< 50`). This ensures that even if a user tweaks their "Top" position to 15% or 20% in `mpv.conf`, the script intelligently recognizes it as the "Top" state and toggles accurately.

### 4. System Key Layout Robustness
Implemented native Russian layout support for standard mpv commands to eliminate the need for manual layout switching.
- **Quit**: Added `й` (q) and `Й` (Q).
- **Mute**: Added `ь` (m).
- **Speed**: Added `х` / `ъ` ([ / ]).
- **Frame Step**: Added `ю` / `б` (. / ,).

## Proposed Changes
- **lls_core.lua**: Added `mp.options`, threshold-based toggling, and safety gap defaults.
- **mpv.conf**: Added `script-opts-append` section for centralized control and documentation.
- **Documentation**: Updated README.md and Release Notes.
