# RFC: Release v1.26.8 (Subtitle Feature Consistency & Feedback)

**Date**: 2026-03-14
**Version**: v1.26.8
**Request ZID**: 20260313235721

## Staging
- [LLS Core](file:///u:/voothi/20260308110646-kardenwort-mpv/scripts/lls_core.lua)

## Problem Description
1. Advanced subtitle features (Drum Mode `c`, Drum Window `w`, Search `Ctrl+f`, and Context Copy `z`) would silently fail or show misleading "ON" states when loaded with embedded subtitles instead of external tracks.
2. "Copy Mode Cycle" (`z`) output was often confusing when only a single subtitle track was available.

## Implementation Details

### Analytics & Decisions
1. **Consistency Audit**:
   - Analyzed the interactions of keybindings (`w`, `search`, `t`, `r`, `j`, `c`, `x`, `z`, `s`) with FSM states (`NO_SUBS`, `SINGLE_SRT`, `DUAL_ASS`, etc.) and identified reliance on external subtitle file loading (`Tracks.pri.path`).
2. **Robust Feature Guarding**:
   - **Action**: Inserted validation checks for `Tracks.pri.path` (and `Tracks.sec.path`) in toggle functions `cmd_toggle_drum`, `cmd_toggle_drum_window`, and `cmd_toggle_search`.
   - **Decision**: Ensures users immediately understand why a feature isn't working if they open a video with only embedded subtitle tracks, preventing perceived silent failures.
3. **Improved Copy Mode Feedback**:
   - **Action**: Expanded `cmd_cycle_copy_mode` OSD messages. It now dynamically reports "A (Primary/Target)" or "B (Secondary/Translation)", and correctly identifies "Fixed to Primary (Single Track)".
   - **Decision**: Replaces cryptic `A` and `B` modes with descriptive labels, significantly reducing onboarding friction for new users managing clipping configurations.

### Verification Results
- **Manual Verification**: Features appropriately display "Requires external subtitle files" when activated on embedded tracks, and correctly activate for external `.srt` and `.ass` (unless explicitly blocked for `.ass` in Drum Mode). Track cycling mode provides extended textual feedback.
